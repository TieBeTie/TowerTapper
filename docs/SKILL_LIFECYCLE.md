# Жизненный цикл навыков в игре

## Типы навыков
1. **Перманентные навыки**: сохраняются на сервере, восстанавливаются в начале каждой игры.
2. **Временные навыки**: существуют только в рамках текущей сессии игры и сбрасываются при завершении игры или перезагрузке страницы.

## Управление навыками

### SkillSetStorage
- Класс SkillSetStorage отвечает за хранение навыков только в памяти во время сессии игры.
- Хранение происходит полностью в памяти, без сохранения в localStorage или другие постоянные хранилища.
- В начале каждой игры хранилище пересоздается с помощью статического метода `recreate()`.

### SkillStateManager
- Менеджер управляет состоянием навыков на протяжении игры.
- При инициализации:
  1. Пересоздает хранилище с помощью `SkillSetStorage.recreate()`
  2. Пытается загрузить перманентные навыки с сервера
  3. Если сервер недоступен, устанавливает базовый уровень 1 для всех перманентных навыков
  4. Сохраняет начальное состояние в хранилище

## Инициализация
1. При старте игры вызывается `SkillStateManager.initialize()`
2. Хранилище пересоздается с помощью `SkillSetStorage.recreate()`
3. Создается пустая карта состояния навыков
4. Вызывается `initializeFromServer()` для загрузки перманентных навыков с сервера:
   - Если сервер доступен и навыки получены, они добавляются в состояние
   - Если сервер недоступен или нет навыков, вызывается `initializeDefaultLevels()` для установки базовых уровней (значение 1)
5. Начальное состояние сохраняется в хранилище

## Во время игры
- Временные навыки могут быть изменены в ходе игры
- Все изменения сохраняются только в памяти
- Метод `resetTemporarySkills()` может быть вызван для сброса только временных навыков

## При окончании игры
- При разрушении башни (метод `Tower.die()`) происходит сброс всех менеджеров:
  1. Вызывается `SkillStateManager.resetOnGameOver()` для очистки временных навыков
  2. Вызывается `UpgradeManager.reset()` для обновления информации о доступных улучшениях
  3. Вызывается `UIManager.reset()` для обновления отображения кнопок улучшений

- Метод `SkillStateManager.resetOnGameOver()`:
  1. Пересоздает хранилище с помощью `SkillSetStorage.recreate()`
  2. Сохраняет только перманентные навыки с сервера
  3. Если сервер недоступен, устанавливает базовый уровень 1 для всех перманентных навыков

- При перезагрузке страницы хранилище пересоздается, и навыки загружаются с сервера заново

## Тестовый режим
- Когда сервер недоступен (например, в тестовом режиме), для перманентных навыков устанавливается базовый уровень 1
- Это позволяет тестировать игру без подключения к серверу

## Как это работает

1. **Локальное хранение**: `SkillSetStorage` хранит все навыки в памяти, но не в `localStorage`, чтобы они не сохранялись между сессиями игры
2. **Серверное хранение**: Перманентные навыки и эмблемы сохраняются на сервере через WebSocket
3. **Сброс временных навыков**: При проигрыше методы `resetOnGameOver()` и `reset()` обеспечивают полное обновление системы навыков

### Перманентные навыки

Следующие навыки считаются перманентными и сохраняются на сервере:

- `SkillType.EMBLEM_BONUS`: Бонус к получаемым эмблемам
- Другие перманентные навыки (добавляйте их в `InitialSkillTypes`)

## Пример использования

```typescript
// Игровой цикл
class Game {
    constructor() {
        this.skillManager = SkillStateManager.getInstance();
        this.skillManager.initialize();
    }
    
    async start(telegramId) {
        if (telegramId) {
            const InitialService = InitialSkillService.getInstance();
            await InitialService.connect(telegramId);
        }
    }
    
    gameOver() {
        // Сбрасываем временные навыки
        this.skillManager.resetOnGameOver();
        
        // Сбрасываем менеджеры улучшений и UI
        this.upgradeManager.reset();
        this.uiManager.reset();
        
        // Дополнительная логика окончания игры
    }
}
```
